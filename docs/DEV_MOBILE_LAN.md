# 모바일에서 PC IP로 개발 서버 접속 (예: `http://172.20.10.2:5173`)

## 새로고침 시 흰 화면만 나올 때

1. **PC에서 dev 서버를 다시 띄운다**  
   `vite.config.ts`에 `server.host: true`가 있어야 같은 Wi-Fi의 폰에서 `http://<PC의 LAN IP>:5173`으로 접속할 수 있습니다.

2. **HMR(핫 리로드) 끄고 시험**  
   터미널에서:
   ```bash
   NO_HMR=1 npm run dev
   ```
   웹소켓(HMR)이 `localhost`로만 붙으면 폰에서는 실패할 수 있어, 새로고침 후 번들이 꼬이는 것처럼 보일 수 있습니다. HMR을 끄면 새로고침 동작이 더 안정적인 경우가 많습니다.

3. **폰 Safari/Chrome에서 “캐시 비우기”**  
   이전에 실패한 스크립트가 캐시된 경우 주소를 다시 치면 되는 것처럼 보일 수 있습니다.

4. **PC 방화벽**  
   포트 `5173` 인바운드가 막혀 있으면 간헐적으로만 로드될 수 있습니다.

## 프로덕션과의 차이

이 증상은 **개발 서버(Vite + HMR)** 에서만 자주 나옵니다. `npm run build` 후 `preview` 또는 실제 배포 URL은 일반 SPA와 동일하게 동작합니다.
