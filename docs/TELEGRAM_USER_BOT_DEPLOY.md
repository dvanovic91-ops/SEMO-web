# Telegram user-бот (личный кабинет): деплой и кнопка «Открыть сайт — редактирование»

## Почему в чате старый вид (ИНН/паспорт без маски, только «Назад»)

В репозитории логика живёт в **`semo_bot/클라우드 작업파일/최종 코드/user_bot.py`**. Если на сервере запущена **старая копия** файла или другой скрипт (`bot.py` и т.д.), в Telegram будет прежний интерфейс.

**Нужно:** залить актуальный `user_bot.py` (+ при необходимости `supabase_helper.py`), перезапустить процесс.

## Переменные окружения (.env рядом с ботом)

| Переменная | Зачем |
|------------|--------|
| `TELEGRAM_USER_BOT_TOKEN` | Токен бота |
| `PUBLIC_SITE_URL` **или** `TELEGRAM_WEB_APP_URL` **или** `WEBSITE_URL` / `BOT_PUBLIC_SITE_URL` / `SEMO_SITE_URL` | Базовый URL сайта с `https://…` (без слэша в конце). **Обязательно на сервере**, иначе кнопка **«🌐 Открыть сайт — редактирование»** не будет `url` (только callback с подсказкой). |
| `PROFILE_EDIT_URL` | (Необязательно) Полный URL страницы редактирования, если отличается от `{PUBLIC_SITE_URL}/profile/edit`. |

**Mini App vs обычная ссылка:** в коде **не используется** `MenuButtonWebApp` / `web_app=` — только `InlineKeyboardButton(url=…)` и `MenuButtonDefault`. Это **не** Telegram Web Apps Mini App. Открытие **внутри Telegram** (in-app) или **в отдельном Safari/Chrome** зависит от клиента и настроек телефона; принудительно «только внешний браузер» API бота не задаёт.

Рядом с важными ссылками — **«📋 Скопировать ссылку → Safari/Chrome»** (`CopyTextButton`), чтобы вставить URL в системный браузер вручную.

Если URL не задан, кнопка всё равно есть, но ведёт на **callback** с подсказкой про `.env`.

При старте бот печатает строку: кнопка URL **OK** или **НЕТ**.

## Проверка после деплоя

1. Личный кабинет → **Мои данные**.
2. ИНН и паспорт по умолчанию **с маской**; есть **«Показать»** / **«Скрыть»** (если в БД заполнены поля).
3. Отдельными рядами: **Открыть сайт — редактирование** → копирование ссылки → **Назад**.
