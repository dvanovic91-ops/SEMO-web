# Telegram-бот: витрина товаров (по одному + навигация)

Код самого бота в этом репозитории **нет** — ниже спецификация для `My_SEMO_Beautybot` (или любого бота на Python/Node).

## Что не так с текущим вариантом

- Сверху длинный текст, снизу альбом фото — пользователь теряет контекст.
- Строки вида **📦 … шт** — это обычно **количество на складе** («штук»). Покупателю **не показываем** остатки.

## Целевой UX

1. Загрузить список товаров **один раз** (см. RPC `telegram_products_catalog` в `docs/SUPABASE_RPC_TELEGRAM_PRODUCTS_CATALOG.sql` — поля **без** `stock`).
2. Показывать **один товар за раз**:
   - Одно сообщение **`sendPhoto`** (первый URL из `image_urls`, иначе `image_url`).
   - **Caption** = порядковый номер, название, очищенное описание, цена. Форматировать как в `src/lib/telegramProductCaption.ts` (`buildTelegramProductCaption` + `sanitizeDescriptionForTelegram`).
3. Под фото — **InlineKeyboard**:
   - Товар **1 / N**: только кнопка **«Следующий товар ▶»** (или «Далее»).
   - Товар **2 … N−1**: **«◀ Предыдущий»** и **«Следующий ▶»**.
   - Товар **N / N**: только **«◀ Предыдущий»**.
4. По нажатию — **редактировать** то же сообщение (`editMessageMedia` + caption) или удалить и отправить новое `sendPhoto` (проще для бота — `deleteMessage` + `sendPhoto`, но чаще используют edit).
5. **Не использовать** `sendMediaGroup` для карусели одного товара — одно фото на шаг; дополнительные картинки товара можно показывать отдельными шагами или только первое фото в MVP.

## Callback data (лимит 64 байта)

Храните у пользователя в памяти сессии `list_index` и используйте короткие callback, например:

- `p:n` — перейти к индексу `n` (0-based), если помещается;
- или `shop_next` / `shop_prev` и хранить индекс в `context.user_data`.

## Про «шт»

- **шт** = **штука** (единица счёта). В сочетании с числом (**12 шт**) почти всегда **остаток на складе** для персонала/админки.
- В текст для клиента **не включать** поле `stock` из БД и **фильтровать** строки описания — см. `sanitizeDescriptionForTelegram` в репозитории.

## Данные из Supabase

После выполнения SQL:

```js
const { data, error } = await supabase.rpc('telegram_products_catalog');
// data: [{ id, name, slug, description, detail_description, rrp_price, prp_price, image_url, image_urls }, ...]
```

`service_role` только на сервере бота, не в клиенте.
