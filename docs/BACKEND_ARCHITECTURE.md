# 뷰티박스 백엔드 — 전체 구조 가이드 (초보자용)

> **목적:** "웹사이트 뒤에서 뭘 돌려야 하지?"를 한 번에 이해할 수 있게, 큰 그림과 필요한 서비스들을 정리한 문서입니다.

---

## 1. 큰 그림: 세 가지 층

```
[사용자]  →  [우리 웹(React)]  →  [백엔드 서버]  →  [데이터베이스]
   브라우저      지금 만든 것          새로 만들 것         회원/주문 저장
```

- **우리 웹(React):** 사용자가 보는 화면. 버튼 누르면 "백엔드 서버"에 요청을 보냅니다.
- **백엔드 서버:** 그 요청을 받아서 로그인 처리, 주문 저장, 이메일 보내기 등을 **대신** 해 줍니다.
- **데이터베이스(DB):** 회원 정보, 주문 내역, 배송 정보 같은 것을 **영구 저장**하는 창고입니다.

**쉽게 말하면:**  
웹은 "창구", 백엔드는 "사무실", DB는 "서류 보관함"이라고 생각하면 됩니다.

## 2. 블록별로 뭘 해야 하는지

### 2-1. 백엔드 서버 (우리 코드가 돌아가는 곳)

| 하는 일 | 설명 | 무료로 가능? |
|--------|------|--------------|
| API 제공 | 로그인, 프로필 저장, 주문 조회 같은 **주소(URL)** 를 만들어 주는 프로그램 | ✅ 가능 |
| 24시간 실행 | 사용자가 언제 들어와도 응답하려면 서버가 계속 켜져 있어야 함 | ⚠️ 조건부 (아래 추천 참고) |

**필요한 것:**  
- 서버를 **올려 둘 컴퓨터** (우리 집 PC가 아니라, 인터넷에 연결된 "호스팅" 서비스)
- **Node.js** 또는 **Python** 등으로 작성한 "API 서버" 코드

---

### 2-2. 데이터베이스 (DB) — 회원·주문 등 저장

| 하는 일 | 설명 | 무료로 가능? |
|--------|------|--------------|
| 회원 정보 저장 | 이메일, 이름, 배송 주소, 비밀번호(암호화) 등 | ✅ 가능 |
| 주문/배송 저장 | 주문 번호, 상품, 배송 상태, 추적 정보 | ✅ 가능 |
| 테스트 결과·리뷰·포인트 | 피부 테스트 결과, 리뷰, 적립 포인트 | ✅ 가능 |

**필요한 것:**  
- **DB 서비스** 하나 (데이터를 파일처럼 저장해 주는 전문 프로그램/서비스)

### 2-3. 로그인 (OAuth — 구글/얀덱스)

| 하는 일 | 설명 | 무료로 가능? |
|--------|------|--------------|
| 구글 로그인 | "구글로 로그인" 클릭 시 구글이 인증해 주고, 우리 서버가 사용자 정보 받음 | ✅ 무료 (할당량 내) |
| 얀덱스 로그인 | "얀덱스로 로그인" 클릭 시 얀덱스가 인증해 주고, 우리 서버가 사용자 정보 받음 | ✅ 무료 |

**필요한 것:**  
- Google Cloud Console / Yandex OAuth 에서 **앱 등록** (무료)
- 백엔드에서 **callback URL** 만들고, `code` 받아서 토큰·이메일 받는 처리

| 비밀번호 재설정 메일 | "비밀번호 잊었어요" 시 링크 발송 | ✅ 같은 서비스로 가능 |

### 2-5. 배포·호스팅 (웹 + API를 인터넷에 올리기)

| 하는 일 | 설명 | 무료로 가능? |
|--------|------|--------------|
| 웹(React) 배포 | 지금 만든 화면을 `https://우리도메인.com` 으로 서비스 | ✅ 무료 플랜 많음 |

- 또는 **웹+API 한 번에** 올릴 수 있는 서비스 1곳

## 3. 서비스별 무료/유료 정리 + 추천

### 3-1. 백엔드 서버 올릴 곳 (호스팅)

| 서비스 | 무료 여부 | 한 줄 설명 | 추천도 |
|--------|-----------|------------|--------|
| **Railway** | ✅ 무료 크레딧 (월 $5 등) | 코드 올리면 자동으로 서버 띄워 줌. DB도 붙이기 쉬움 | ⭐⭐⭐ 처음에 쓰기 좋음 |
| **Render** | ✅ 무료 플랜 있음 (슬립 모드 있음) | GitHub 연결해서 자동 배포. 소규모에 적합 | ⭐⭐⭐ |
| **Fly.io** | ✅ 무료 티어 있음 | 컨테이너로 배포. 약간 설정이 많음 | ⭐⭐ |
| **Vercel** | ✅ 무료 (서버리스) | 프론트는 최적. 백엔드 API는 "서버리스 함수"로만 가능 (DB 연결 등 제한 있음) | ⭐ 프론트 배포용으로만 |
| **AWS / GCP / Azure** | ⚠️ 무료 티어 있지만 과금 실수 시 비용 발생 | 대형 인프라. 초보자는 설정·과금이 부담 | ⭐ 나중에 규모 키울 때 |

**의견·추천:**  
- **시작 단계:** **Railway** 또는 **Render** 중 하나 추천.  
  - GitHub에 코드 올리고 "이 레포 연결해서 배포해 줘"만 하면 돼서 초보자도 따라 하기 쉽습니다.  
  - Railway는 DB(PostgreSQL)를 같은 대시보드에서 한 번에 붙일 수 있어서 더 편할 수 있습니다.

---

### 3-2. 데이터베이스 (DB)

| 서비스 | 무료 여부 | 한 줄 설명 | 추천도 |
|--------|-----------|------------|--------|
| **Supabase** | ✅ 무료 플랜 (500MB 등) | PostgreSQL + 로그인/API 자동 생성. 실시간 기능도 있음 | ⭐⭐⭐ |
| **PlanetScale** | ✅ 무료 플랜 (MySQL) | MySQL 호스팅. 브랜치로 DB 관리 가능 | ⭐⭐⭐ |
| **Railway PostgreSQL** | ✅ Railway 크레딧으로 사용 | Railway 쓰면 같은 곳에서 DB 생성 가능 | ⭐⭐⭐ |
| **MongoDB Atlas** | ✅ 무료 클러스터 (512MB 등) | 문서형 DB. JSON처럼 저장하고 싶을 때 | ⭐⭐ |
| **Neon** | ✅ 무료 티어 (PostgreSQL) | 서버리스 PostgreSQL. cold start 있을 수 있음 | ⭐⭐ |

**의견·추천:**  
- **PostgreSQL** 쓰는 걸 추천합니다. (회원, 주문, 배송 같은 **표 형태** 데이터에 잘 맞음.)  
- **Railway**로 백엔드 올릴 거면 → **Railway에서 PostgreSQL 추가**가 제일 간단합니다.  
- "백엔드는 최소한으로 하고, DB+간단 API를 빠르게 쓰고 싶다"면 **Supabase**도 좋습니다. (가입/로그인, 테이블, REST API를 웹에서 설정만으로 쓰는 느낌.)

---

### 3-3. 이메일 발송 (Support 문의 등)

| 서비스 | 무료 여부 | 한 줄 설명 | 추천도 |
|--------|-----------|------------|--------|
| **Resend** | ✅ 월 3,000통 무료 | API로 메일 발송. 설정 간단, 문서 좋음 | ⭐⭐⭐ |
| **SendGrid** | ✅ 월 100통 무료 | 오래된 서비스. 무료 한도 적음 | ⭐⭐ |
| **Mailgun** | ✅ 제한적 무료 | 개발용으로 괜찮음 | ⭐⭐ |
| **AWS SES** | ✅ 티어 내 무료 (넉넉) | AWS 쓰는 경우. 설정이 다소 복잡 | ⭐ AWS 쓸 때만 |

**의견·추천:**  
- **Resend** 추천.  
  - 무료 한도가 넉넉하고, "이메일 주소로 API 키 받아서 한 줄 호출" 수준으로 쓰기 쉽습니다.  
  - Support 폼에서 "문의 내용 받아서 우리 메일로 전달"만 하면 되므로 Resend 한 개로 충분합니다.

### 3-4. OAuth (구글 / 얀덱스 로그인)

| 구분 | 무료 여부 | 비고 |
|------|-----------|------|
| **Google OAuth** | ✅ 무료 (할당량 내) | Google Cloud Console에서 프로젝트·OAuth 클라이언트 생성 |
| **Yandex OAuth** | ✅ 무료 | Yandex OAuth에서 앱 등록 후 Callback URI 설정 |

**의견:**  
- 둘 다 **추가 비용 없이** 사용 가능.  
- 백엔드에서 할 일은 "callback URL 만들기 + code로 토큰 받기 + 이메일/이름 저장"입니다. (이미 `OAUTH_SETUP.md`에 정리돼 있음.)

### 3-5. 웹(프론트) 배포

| 서비스 | 무료 여부 | 한 줄 설명 | 추천도 |
|--------|-----------|------------|--------|
| **Vercel** | ✅ 무료 | React/Vite 빌드해서 배포하기 매우 쉬움 | ⭐⭐⭐ |
| **Netlify** | ✅ 무료 플랜 | Vercel이랑 비슷. 폼·리다이렉트 설정 편함 | ⭐⭐⭐ |
| **Cloudflare Pages** | ✅ 무료 | 빠른 CDN. 설정 약간 더 있음 | ⭐⭐ |

**의견·추천:**  
- 지금처럼 **React + Vite** 프로젝트는 **Vercel**에 올리는 게 제일 빠릅니다.  
  - GitHub 연결 → `npm run build` 자동 실행 → 배포 끝.  
- 도메인은 나중에 "우리 도메인 연결"만 해 주면 됩니다.

## 4. 조합 추천 (처음 백엔드 붙일 때)

| 용도 | 추천 조합 | 이유 |
|------|-----------|------|
| **가장 간단하게 시작** | **Railway** (백엔드 + PostgreSQL) + **Vercel** (웹) + **Resend** (이메일) | 서비스 수 적고, 가입·연결만으로 대부분 해결 |
| **백엔드 코드 최소화** | **Supabase** (DB + Auth + API) + **Vercel** (웹) + **Resend** | DB와 인증을 Supabase가 많이 대신해 줌. 우리는 OAuth callback·비즈니스 로직만 |
| **비용 0 유지** | **Render** (백엔드) + **Neon** 또는 **Supabase** (DB) + **Vercel** (웹) + **Resend** | 전부 무료 플랜으로 1년 이상 운영 가능 (트래픽 적을 때) |

**정리:**  
- **"초보자도 쉽게 + 한 곳에서 관리하고 싶다"** → **Railway(백엔드+DB) + Vercel(웹) + Resend(메일)**  
- **"백엔드 코드를 최대한 안 쓰고 싶다"** → **Supabase 중심 + Vercel + Resend**

## 5. 우리 프로젝트에서 필요한 API (체크리스트)

백엔드를 만들 때 "이런 주소(API)가 있으면 된다" 목록입니다.

| API (주소) | 메서드 | 하는 일 | 우선순위 |
|------------|--------|---------|----------|
| `/auth/google/callback` | GET | 구글 로그인 후 code 받아서 회원 생성/로그인 | 🔴 필수 |
| `/auth/yandex/callback` | GET | 얀덱스 로그인 후 code 받아서 회원 생성/로그인 | 🔴 필수 |
| `/me` 또는 `/me/profile` | GET | 로그인한 회원 정보 조회 | 🔴 필수 |

| `/orders` | GET | 주문 목록 (히스토리) | 🟠 권장 |
| `/orders/:id` | GET | 주문 상세 + 배송 추적 | 🟡 나중에 |

| `/me/reviews` | GET | 내 리뷰 목록 | 🟡 나중에 |
| `/me/points` | GET | 포인트 잔액·내역 | 🟡 나중에 |

## 6. 다음 단계 (실행 순서 제안)

1. **백엔드 프로젝트 정하기**  
   - Node.js (Express/Fastify) 또는 Python (FastAPI) 등으로 "API 서버" 레포 하나 만든다.
2. **DB 하나 정해서 연결**  
   - 예: Railway PostgreSQL 또는 Supabase.
3. **테이블 설계**  
   - `users`, `shipping_addresses`, `orders` 정도부터 만든다.
4. **OAuth callback 구현**  
   - 구글 → callback → 토큰 교환 → 이메일로 회원 생성/조회 → JWT 또는 세션 발급.
5. **프론트 `.env`에 API 주소 넣기**  
   - 예: `VITE_API_URL=https://api.우리도메인.com`
6. **Support 이메일 연동**  
   - Resend 가입 → API 키 받아서 `/support` POST 처리에서 메일 전송.
7. **배포**  
   - API는 Railway/Render, 웹은 Vercel에 올린 뒤, 도메인·CORS·OAuth Redirect URI만 최종 수정.

## 7. 보안·확장성 우선 (비용 어느 정도 감수해도 괜찮을 때)

편의성·안전성·보안·확장성을 우선하고, 비용이 들더라도 괜찮다면 아래 구성을 추천합니다.

### 7-1. 왜 비용을 조금 더 쓰는가

| 목표 | 무료/저비용일 때 | 비용을 쓰면 얻는 것 |
|------|------------------|----------------------|
| **보안** | 단일 서버, 공개 DB 접속 등 | 관리형 DB(암호화·백업), 전용 네트워크(VPC), WAF·DDoS 방어 |
| **확장성** | 트래픽 한도, 슬립 모드, cold start | 항상 켜진 서버, 오토스케일, 리전 확장 |
| **편의** | 여러 서비스 조합, 수동 백업 | 한 플랫폼에서 DB·백업·모니터링 통합 |
| **안정성** | 무료 티어 제한, 장애 시 지원 제한 | SLA, 자동 복구, 24시간 지원 |

### 7-2. 보안·확장성 추천 조합

| 구분 | 추천 서비스 | 월 비용 대략 | 이유 |
|------|-------------|--------------|------|
| **백엔드 호스팅** | **Railway** (유료) 또는 **Render** (유료 플랜) | $5~20 | 슬립 없음, 항상 실행, DB와 같은 플랫폼에서 관리 |
| **DB** | **Supabase Pro** 또는 **Neon** Scale / **AWS RDS** | $25 전후 ~ | 암호화(저장·전송), 자동 백업, 포인트인타임 복구 |
| **이메일** | **Resend** (유료 플랜) 또는 **SendGrid** (Pro) | $20 전후 ~ | 발송 한도·전용 IP·도메인 인증으로 스팸 방지·안정성 |
| **웹 배포** | **Vercel Pro** (선택) | $20 전후 | 비공개 팀, 더 넉넉한 빌드·대역폭, 지원 |
| **도메인·SSL** | **Cloudflare** (무료/Pro) | $0~20 | DNS·SSL 무료, Pro 시 WAF·DDoS 완화·보안 규칙 |

### 7-3. 구체적 추천 (한 줄 요약)

- **“한 곳에서 다 관리하고, 보안·백업은 믿을 수 있게”**  
  → **Railway(백엔드 + PostgreSQL 유료)** + **Supabase Pro(DB만 쓸 경우)** 또는 **Railway PostgreSQL(백업·암호화 확인)** + **Resend 유료** + **Vercel(웹)** + **Cloudflare(도메인·SSL)**  
  - DB는 암호화·자동 백업이 되는 플랜을 쓰고, API 키·비밀은 환경 변수로만 관리합니다.

- **“AWS/GCP 수준의 보안·확장성을 원한다”**  
  → **AWS**: **ECS/Fargate** 또는 **App Runner**(백엔드) + **RDS PostgreSQL** + **SES**(이메일) + **CloudFront + S3**(웹)  
  → **GCP**: **Cloud Run**(백엔드) + **Cloud SQL** + **SendGrid/SES** + **Firebase Hosting** 또는 **Cloud CDN**  
  - 비용은 월 $50~150 이상 나올 수 있지만, VPC·IAM·암호화·백업·리전 확장을 제대로 쓸 수 있습니다.

- **“비용은 적당히, 보안만 올리고 싶다”**  
  → **Render(Railway) 유료** + **Supabase Pro** 또는 **Neon** + **Resend** + **Cloudflare(무료)**  
  - 무료보다 한 단계 위 플랜만 써도 백업·암호화·항상 켜진 서버로 안정성과 보안이 크게 올라갑니다.

### 7-4. 보안·확장성 체크리스트

- [ ] DB: **저장 암호화(at rest)** + **전송 암호화(TLS)** 사용
- [ ] DB: **자동 백업** 및 필요 시 **포인트인타임 복구** 가능한 플랜
- [ ] API: **HTTPS만** 사용, **CORS**는 우리 웹 도메인만 허용
- [ ] 비밀·API 키: **환경 변수**에만 저장, 코드/저장소에 절대 넣지 않기
- [ ] 로그인: **JWT** 또는 **세션** 만료 시간 짧게, 필요 시 **Refresh Token** 사용
- [ ] 이메일: **발송 도메인 인증**(SPF/DKIM)으로 스팸·피싱 위험 줄이기
- [ ] (선택) **WAF·DDoS 방어**(Cloudflare Pro 등)로 API·웹 보호

비용이 들더라도 **DB 백업·암호화**, **항상 켜진 서버**, **이메일 도메인 인증**까지 해 두면 편하고 안전하게 서비스를 키울 수 있습니다.
